About Backup

Espresso ELN provides infrastructure for securing your experiment data to prevent data loss, e.g. after a hardware failure or after migrating to a new machine. There are two protection layers:
 
First layer: Espresso ELN offers automatic data synchronization to a backup destination, where all changes to your experiments database are updated every 10 minutes, either to a backup copy of your experiments database, or to a dedicated in-house and open source MySQL or MariaDB server database. It is even possible to switch between file-based and server-based backup, since two separate synchronization destination tracks are maintained. The server database additionally serves as a continuously growing in-house reaction database; all Espresso ELN clients are able to use this data sharing functionality by switching the search scope from Local to Global in the search section.


All backup functionality is available in the Dashboard Section of the application:



 A backed up experiment database can be restored in a few simple steps.

Second layer: From version 6 on, the experiments database is stored in your personal documents folder (under the sub-folder "Espresso ELN Data"); if you regularly back up your personal documents, or your organization does this for you, your experiments database will also be backed up along with them.
