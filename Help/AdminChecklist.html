Admin Checklist

User Migration To New PC

When preparing a new machine for an existing Espresso ELN user, please follow the steps below:

  • Install the latest downloadable Espresso ELN demo version (download page). 
  • Replace the demo experiments database by the original experiments database of the user. This is performed using the Restore from Server functionality, which also keeps the user's license intact and no re-activation is required. Restoring from the server requires the user password.
  • If restoring from the server should not be possible for any reason, or the password is unknown, then the demo database located at 'MyDocuments\Espresso ELN Data\espressoData.db' also can be manually replaced by the user's original database of the same name. If not available anymore, there are chances that this file may be part of a daily backup of the user's personal data folder.

User Fluctuations

Sometimes a user of an Espresso ELN installation leaves the department or company, and his successor needs to take over his license. There are two ways to achieve this without requiring an additional activation:

Site Licenses

If a site license in place, and your Espresso ELN clients are version 10.1.3 and higher, then the process is simple:

  • Deactivate the license of the leaving user in the 'Server Users' section of ELN Admin Tool.
  • Download and install the latest Espresso ELN demo on the PC of the new user and activate it.

Single Licenses

If no site license is in place, the local experiments database containing the license information must be kept and the new user must be added to its list of local users:

  • If the new user obtains a newly installed PC, follow the steps outlined in above 'User Migration' section for restoring the previous user's experiments database from the server.
  • When done, or if the new user just continues to utilize the PC of the previous user, then there are two options:
    • Create a new local user for the new user (Create New User topic). This keeps the data of both users on the  local database, which can be toggled using the users dropdown menu above the experiments tree. 
    • Or: Continue with the userID of the previous user, but replace his user settings information with your own data (Home Section -> Settings -> User Settings). The new name now will appear on the new experiment headers, but under the previous UserID. 

Reset User Password

If a user has forgotten his password, it can be reset from the Server Users Section of ELN Admin Tool. When restarting his client (and connected to the ELN server), the entry of a new password is automatically requested (from client version 8.5.2 on).

New Installations

When creating their first non-demo user, users are required to enter their name, department, site and company information, which serve as header info for their experiments. Please make sure that especially the 'City' information is identical for all users of the same site to allow meaningful filtering by site in the Server Users Section of ELN Admin Tool.
